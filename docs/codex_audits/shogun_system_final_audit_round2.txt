**Axis 6 Implementation Plan**

Add `shim/hakudokai/hakudokai_git_guard.py` or `.sh` with subcommands:

- `prework`: `git fetch --prune`, resolve `@{u}` dynamically, refuse hard-coded `master`; if clean and behind-only, `git merge --ff-only`; if dirty/behind/diverged, send alert and stop.
- `wip-commit`: every 60 min, if dirty, run `git diff --check`, stage with deny patterns for secrets, commit `[WIP] <task_id> <pc_id> <timestamp>`; never push automatically.
- `check-divergence`: `git rev-list --left-right --count HEAD...@{u}`; if ahead>0 and behind>0, send Supabase `urgent_stop`.
- `check-unpushed`: every 5 min, if `git log @{u}..HEAD` non-empty for more than 5 min, send high-priority warn alert.
- `prepush`: installed as `.git/hooks/pre-push`; fetch first, block push if remote advanced/diverged.

Integration points:

- `shutsujin_departure.sh`: call `git_guard prework` after `cd "$SCRIPT_DIR"`.
- `shim/hakudokai/hakudokai_departure.sh`: call `git_guard prework` before watcher start.
- `shim/hakudokai/hakudokai_watchdog.sh`: call `wip-commit` hourly and `check-unpushed/check-divergence` every 5 min.
- `scripts/install_git_hooks.sh`: installs pre-push hook.
- `config/settings.yaml`: add `git_guard` config: enabled, remote, branch/upstream mode, WIP interval, alert role, strict mode.

Estimated size: `git_guard` 300-350 LOC, hook installer 50-70 LOC, watchdog/startup integration 40-60 LOC, tests 120-180 LOC. Total: roughly **550 LOC**.
